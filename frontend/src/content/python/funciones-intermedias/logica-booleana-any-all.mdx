# 4.4 Lógica booleana global con any() y all()

Las funciones `any()` y `all()` permiten evaluar condiciones sobre **colecciones enteras** de una sola vez. Son herramientas esenciales para validación, filtrado y toma de decisiones en programas reales.

---

### Definición

## any()

Devuelve `True` si **al menos uno** de los elementos es truthy. Devuelve `False` solo si **todos** son falsy:

### Ejemplos básicos

<CodeRunner language="python" title="any() — ejemplos básicos" initialCode={`# Al menos un elemento truthy → True
print(any([False, False, True]))    # True
print(any([0, 0, 1]))              # True
print(any([None, "", "hola"]))     # True

# Todos falsy → False
print(any([False, False, False]))   # False
print(any([0, 0, 0]))              # False
print(any([]))                      # False (vacío)`} />

### Listas booleanas

<CodeRunner language="python" title="any() con listas booleanas" initialCode={`resultados_examen = [True, False, True, True, False]

todos_suspendidos = [False, False, False]
print(f"¿Alguien aprobó? {any(todos_suspendidos)}")    # False`} />

### Strings

<CodeRunner language="python" title="any() con strings" initialCode={`# Las cadenas vacías son falsy, las no vacías son truthy

print(any(campos))    # True — al menos un campo tiene valor

print(any(campos_vacios))   # False — todos vacíos`} />

### Condiciones

<CodeRunner language="python" title="any() con condiciones" initialCode={`numeros = [2, 4, 6, 7, 8]

## all()

Devuelve `True` si **todos** los elementos son truthy. Devuelve `False` si **al menos uno** es falsy:

### Ejemplos básicos

<CodeRunner language="python" title="all() — ejemplos básicos" initialCode={`# Todos truthy → True
print(all([True, True, True]))     # True
print(all([1, 2, 3]))             # True
print(all(["a", "b", "c"]))      # True

# Al menos un falsy → False
print(all([True, False, True]))    # False
print(all([1, 0, 3]))             # False

# Vacío → True (vacuamente verdadero)
print(all([]))                     # True`} />

### Validación completa

<CodeRunner language="python" title="all() — validación completa" initialCode={`def formulario_completo(campos):
    """Verifica que TODOS los campos tengan valor."""
    return all(campos.values())

form_ok = {"nombre": "Ana", "email": "ana@mail.com", "edad": "28"}
form_mal = {"nombre": "Ana", "email": "", "edad": "28"}

print(f"Completo: {formulario_completo(form_ok)}")    # True
print(f"Completo: {formulario_completo(form_mal)}")   # False`} />

## Comparación directa

<CodeRunner language="python" title="Comparación any() vs all()" initialCode={`# Caso especial: lista con un truthy entre ceros
print(f"any([0, 0, 1]) = {any([0, 0, 1])}")    # True — hay un truthy
print(f"all([0, 0, 1]) = {all([0, 0, 1])}")    # False — no todos son truthy

# Caso especial: lista vacía
print(f"any([])       = {any([])}")              # False — no hay ningún truthy
print(f"all([])       = {all([])}")              # True  — vacuamente verdadero

# Caso: todos truthy
print(f"any([1, 2, 3]) = {any([1, 2, 3])}")    # True
print(f"all([1, 2, 3]) = {all([1, 2, 3])}")    # True

# Caso: todos falsy
print(f"any([0, 0, 0]) = {any([0, 0, 0])}")    # False
print(f"all([0, 0, 0]) = {all([0, 0, 0])}")    # False`} />

> Usa **expresiones generadoras** `(x for x in ...)` en lugar de listas `[x for x in ...]` dentro de `any()` y `all()`. Son más eficientes porque `any()` se detiene en el primer `True` y `all()` en el primer `False`, sin evaluar el resto.
