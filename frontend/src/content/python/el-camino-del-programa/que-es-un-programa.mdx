# 1.1 ¿Qué es un programa?

Un **programa** es una secuencia de instrucciones que le indica al ordenador cómo realizar una tarea. Esa tarea puede ser matemática (resolver una ecuación), simbólica (buscar y reemplazar texto) o visual (procesar una imagen). En todos los casos, el programa descompone un problema complejo en pasos simples que el ordenador puede ejecutar.

## Un modelo mental: tipos de instrucciones

A nivel conceptual, los programas pueden describirse como combinaciones de cinco tipos de instrucciones. Este modelo no forma parte de la sintaxis de Python, sino que es una forma útil de clasificar lo que hace el código. Aunque los programas reales pueden ser más complejos, muchos se ajustan a esta clasificación:

## Flujo de control

Por defecto, Python ejecuta las instrucciones de arriba a abajo, en el orden en que aparecen en el código. Este orden se denomina **flujo de control**. Las instrucciones condicionales y de repetición pueden modificarlo, haciendo que ciertas instrucciones se ejecuten o se repitan en función de condiciones. El ordenador aplica cada instrucción exactamente como está escrita, sin inferir intenciones más allá de lo que el código especifica.

## Ejemplo: ejecución secuencial

El caso más simple es una secuencia de instrucciones que se ejecutan una tras otra, en el orden en que aparecen:

Cada línea se ejecuta en orden. Cambiar el orden de las instrucciones cambia el resultado del programa.

## La analogía de la receta

Un programa puede compararse con una receta de cocina: ambos describen una secuencia de pasos para obtener un resultado a partir de unos datos de entrada.

<table>
  <thead>
    <tr>

  </thead>
  <tbody>
    <tr><td>Ingredientes</td><td>Datos de entrada</td></tr>
    <tr><td>Pasos numerados</td><td>Instrucciones en orden</td></tr>
    <tr><td>"Si no tienes X, usa Y"</td><td>Instrucción condicional</td></tr>
    <tr><td>"Repite durante 5 minutos"</td><td>Instrucción de repetición</td></tr>

A diferencia de una receta, que puede interpretarse con cierto margen de flexibilidad, un programa se ejecuta de forma determinista: dadas las mismas condiciones de entrada, produce siempre el mismo resultado.
