# 3.2 Sintaxis y estructura

Una función en Python tiene una estructura fija y bien definida. Conocer cada parte y su propósito es la base para escribir funciones correctas y legibles.

---

## La palabra clave def

Toda definición de función comienza con `def`, seguida del nombre, los paréntesis y dos puntos:
```python
def nombre_función():
    # cuerpo
```

<CodeRunner language="python" title="Estructura básica" initialCode={`def saludar():
    print("Hola, mundo")

saludar()`} />

## Indentación

La indentación define qué instrucciones pertenecen al cuerpo de la función. En Python se usan **4 espacios** por nivel. Una indentación incorrecta produce un `IndentationError`:

<CodeRunner language="python" title="Indentación correcta e incorrecta" initialCode={`# ✅ Correcto: todas las líneas del cuerpo indentadas al mismo nivel
def saludar():
    print("Hola")
    print("¿Qué tal?")

saludar()

# Esta línea está fuera de la función (no indentada)
print("Esto siempre se ejecuta")`} />

<CodeRunner language="python" title="IndentationError" initialCode={`# ❌ Indentación inconsistente -> IndentationError
try:
    exec("""
def funcion_mal():
    print("línea 1")
  print("línea 2")
""")
except IndentationError as e:
    print(f"IndentationError: {e}")`} />

> Usa siempre **4 espacios** por nivel de indentación. No mezcles espacios y tabuladores: aunque visualmente parezcan iguales, Python los trata como distintos y puede producir errores difíciles de detectar.

---

## Cuerpo vacío: pass

Una función no puede tener el cuerpo completamente vacío. Si necesitas definir una función sin implementarla todavía, usa `pass` como marcador:

<CodeRunner language="python" title="Función con pass" initialCode={`def funcion_pendiente():
    pass   # marcador: indica cuerpo vacío intencionadamente

funcion_pendiente()   # no hace nada, pero no da error`} />

---

## Ejecución: definición vs llamada

Cuando Python lee una definición `def`, **no ejecuta el cuerpo**: simplemente registra la función y continúa. El cuerpo se ejecuta únicamente cuando se llama a la función:

<CodeRunner language="python" title="Definición vs ejecución" initialCode={`print("1 — antes de la definición")

def saludar():
    print("   -> dentro de saludar()")

print("2 — después de la definición")

saludar()   # aquí se ejecuta el cuerpo

print("3 — después de la llamada")`} />

El orden de salida es `1 -> 2 -> dentro -> 3`. La definición no produce ninguna salida.

Las funciones siguen las mismas convenciones de nombre que las variables, con una recomendación adicional: los nombres de función deben ser **verbos o frases verbales** que describan la acción que realizan:
